TEC Talent Pool

A comprehensive Talent Pool Management System for TEC's in-house graduates, built with HTML, CSS, JavaScript, and Cloudflare Workers/Pages. Fully cloud-based with no local database dependencies.

Cloudflare Pages Cloudflare D1 Cloudflare R2 Cloudflare Workers PBKDF2 GitHub CI/CD

Overview

TEC Talent Pool Management System is a comprehensive platform designed to manage and showcase TEC's in-house graduate talent. Built entirely on Cloudflare's infrastructure, this fully cloud-based system requires no local database dependencies. All data is stored on Cloudflare D1 Database, images on Cloudflare R2, and compute runs on Cloudflare Workers/Pages. The system features automatic deployments via GitHub integration, CI/CD pipelines, and preview deployments for pull requests.

Architecture Overview

The system follows a cloud-only architecture with all services running on Cloudflare's edge network. The frontend is hosted on Cloudflare Pages, API endpoints run on Cloudflare Workers, data is stored in Cloudflare D1, and media files are stored in Cloudflare R2. Bindings are automatically configured via wrangler.toml, eliminating the need for manual configuration.

Request Flow

  • Public Candidate View: index.html ‚Üí Cloudflare Pages ‚Üí API endpoint ‚Üí D1 Database ‚Üí Display candidates
  • Candidate Profile: candidate.html?id=X ‚Üí API endpoint ‚Üí D1 Database ‚Üí Display profile
  • Admin Login: admin.html ‚Üí /api/admin/login ‚Üí Token validation ‚Üí Dashboard access
  • Image Upload: Drag & Drop ‚Üí /api/upload ‚Üí R2 Storage ‚Üí URL Returned
  • Image Serving: /api/images/* ‚Üí R2 Storage ‚Üí Display image

API Endpoints

Public Endpoints

  • GET /api/candidates - Get all candidates
  • GET /api/candidates/:id - Get a specific candidate
  • GET /api/notifications - Get all notifications
  • GET /api/images/* - Serve images from R2 storage

Protected Endpoints (Require Admin Token)

Candidate Management

  • POST /api/candidates - Create a new candidate
  • PUT /api/candidates/:id - Update a candidate
  • DELETE /api/candidates/:id - Delete a candidate

Image Management

  • POST /api/upload - Upload image to R2 (profile pictures, certificates)

Admin Management

  • POST /api/admin/login - Admin login
  • GET /api/admin/verify - Verify admin token
  • POST /api/admin/change-password - Change admin password
  • GET /api/admin/admins - List all admins
  • POST /api/admin/admins - Create new admin
  • PUT /api/admin/admins/:id - Update admin
  • DELETE /api/admin/admins/:id - Delete admin

Notification Management

  • DELETE /api/notifications - Clear all notifications
  • DELETE /api/notifications/:id - Delete single notification

Getting Started

Prerequisites

  • Node.js and npm
  • Cloudflare account with access to D1, R2, and Pages
  • Wrangler CLI (npm install -g wrangler)
  • GitHub account (for CI/CD integration)

Setup Instructions

1. Install Dependencies

npm install

2. Configure Cloudflare Services

# Authenticate with Cloudflare
npx wrangler login

# Verify authentication
npx wrangler whoami

# Create D1 database (if not exists)
npx wrangler d1 create tec-talent-pool-db

# Create R2 bucket (if not exists)
npx wrangler r2 bucket create tec-talent-pool-images

# Migrate database schema to remote D1 database
npm run migrate

# Create admin account in remote database
npm run add-admin admin YourSecurePassword123!

# Verify setup
npm run verify

3. Connect to GitHub (Recommended)

# Push code to GitHub
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/your-username/your-repo-name.git
git push -u origin main

Then connect GitHub to Cloudflare Pages in the Cloudflare Dashboard. Bindings are automatically configured from wrangler.toml - no manual setup needed!

4. Development Server

# Start development server (connects to remote Cloudflare D1 database)
npm run dev

# Clean restart (if issues)
npm run dev:clean

Note: The development server connects directly to the remote Cloudflare database. All data operations use cloud services. You need an internet connection.

Challenges & Solutions

Developing TEC Talent Pool involved several technical challenges to ensure a secure, scalable, and cloud-native application.

Challenge: Cloud-Only Architecture

Building a fully cloud-based system with no local database dependencies while maintaining development efficiency.

Solution

  • Used Cloudflare D1 for remote database storage
  • Configured development server to connect directly to remote services
  • Managed all bindings via wrangler.toml for consistency
  • Implemented automatic deployments via GitHub integration
  • Created preview deployments for testing

Result: Fully cloud-based system with seamless development workflow.

Challenge: Image Management

Handling image uploads, storage, and serving in a serverless environment.

Solution

  • Used Cloudflare R2 for object storage
  • Created upload endpoint with validation
  • Implemented image serving via Cloudflare Pages Functions
  • Organized images by type (profile, certificate) in R2 buckets
  • Added proper CORS configuration for image access

Result: Efficient image management with CDN benefits.

Challenge: Secure Authentication

Implementing secure password storage and token-based authentication in a serverless environment.

Solution

  • Used PBKDF2 with SHA-256 for password hashing
  • Implemented password strength validation
  • Created token-based authentication system
  • Added token verification on protected endpoints
  • Implemented secure password change functionality

Result: Secure authentication system with industry-standard practices.
