Overview

Note.Lab is a cloud-based, developer-centric note-taking platform crafted to support technical workflows. Inspired by the clean and minimal aesthetics of GitHub, it offers a familiar environment where developers can focus on writing and organizing their ideas without distractions. With its powerful Markdown editor and built-in syntax highlighting, Note.Lab is ideal for documenting project plans, storing reusable code snippets, managing credentials, drafting technical documents, and keeping track of development notes ‚Äî all in one unified workspace.

Technical Stack

Frontend

  • HTML, CSS, JavaScript - Core web technologies
  • Markdown editor - Powered by CodeMirror
  • Syntax highlighting - Via Prism.js

Backend (serverless)

  • Cloudflare Workers - API endpoints & business logic
  • Cloudflare D1 (SQLite) - Database with full-text search support
  • Cloudflare R2 - Object storage for file attachments
  • Cloudflare KV - Key-value store for session management / caching
  • itty-router - Lightweight router
  • JWT - Authentication using JSON Web Tokens

Getting Started (for developers)

Prerequisites

  • Node.js (v16+) and npm
  • A Cloudflare account with permissions to create D1, R2, KV, and Cloudflare Workers
  • Wrangler CLI (npm install -g wrangler)

Setup & Local Dev

# 1. Clone the repo
git clone https://github.com/Aeroer-Live/Note.Lab.git
cd Note.Lab

# 2. Install dependencies
npm install

# 3. Login to Cloudflare using Wrangler
wrangler login

# 4. Create Cloudflare D1 database
wrangler d1 create note-lab-db
# then update wrangler.toml with the DB ID and run schema:
wrangler d1 execute note-lab-db --file=backend/schema/schema-v2.sql

# 5. Create KV namespace (e.g. "SESSIONS") and R2 bucket 
# (e.g. "note-lab-attachments"), then update wrangler.toml accordingly

# 6. Add secrets
wrangler secret put JWT_SECRET
wrangler secret put ENVIRONMENT

# 7. Start frontend and backend for development
npm run serve        # frontend
wrangler dev         # backend (in a separate terminal)

Deploy to Production

# Deploy backend
wrangler deploy

# Build frontend and deploy (upload `dist/` to hosting provider)
npm run build

Challenges & Solutions

Developing Note.Lab involved navigating several technical challenges to ensure a fast, secure, and developer-friendly experience. Below are key challenges encountered during development and the solutions implemented to overcome them.

Challenge: Ensuring Fast Performance on a Serverless Architecture

Cloudflare Workers offer high performance, but because they are stateless and distributed globally, designing fast, predictable API responses can be difficult ‚Äî especially when handling large notes, attachments, or frequent queries.

Solution

  • Implemented lightweight routing with itty-router to reduce overhead
  • Optimized D1 SQL queries using proper indexing (specifically for title, tags, and updatedAt)
  • Cached frequently used data using Cloudflare KV to reduce database hits
  • Split large operations (such as note fetching, searching, and handling attachments) into dedicated endpoints

Result: Achieved fast, consistent performance across regions with minimal latency.

Challenge: Secure Authentication Without a Traditional Server

Implementing secure user authentication on a serverless runtime requires careful handling of sessions, tokens, and secrets, as there is no persistent server to store sessions.

Solution

  • Adopted JWT-based authentication for stateless login
  • Used KV storage to manage session tokens and enforce their expiry
  • Added token signing with JWT_SECRET and allowed for environment separation (e.g., ENVIRONMENT=prod/dev)
  • Implemented protections such as token invalidation on logout and IP/device checks

Result: Stable, secure, serverless-compatible authentication.
